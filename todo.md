# 项目未来优化方向（待实施）

以下为已建议但尚未落地的优化项与可选增强，按优先级归档，便于后续迭代接续实施。

## 高优先级
- 408 并发一致性（乐观锁）：订单状态更新在 SQL 层加入“期望状态”条件，避免后写覆盖前写；界面提示冲突并刷新最新状态。
- 端到端验收（412）：完成从商家处理订单到消费者查看状态的闭环演示与 Demo 录制。
- DAO/Servlet 集成测试（407）：分页/筛选/详情/更新与会话校验覆盖，提升稳定性与回归效率。
- 文档与测试页（411）：更新 API.md 与 log.md，同步补充 `webapp/test/api-tests.jsp` 商家端用例。

## 中优先级
- 历史订单筛选控件：在前端加入状态/时间范围筛选与分页导航；后端已支持参数与响应头（`X-Page/X-Size`）。
- 菜单签名差异可视化：在再次购买或菜单变更时展示差异项（`perPackageQuantity/sortOrder/price`），引导用户确认。
- 厨房面板完善（405）：核对分栏与快捷操作体验，SSE 失败时回退到手动刷新/轮询策略。
- 路由与鉴权复核（406/410）：逐页核查 `/admin/*` 的路由注册与数据隔离，形成审计结论清单。

## 低优先级

- 错误提示完善：前端统一拦截器与组件化提示（含 401/409/5xx）。
