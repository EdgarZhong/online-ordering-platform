### 前端UI/UX设计文档

**目标:** 一个**状态感知 (State-Aware)** 的多页面在线订餐平台网页应用。应用的核心体验会根据用户是否存在“进行中”的订单而动态改变。

**应用名称:** Forkly

**核心功能:** 用户可以发现餐厅、浏览菜单、在线下单，并能清晰地追踪“进行中”的订单状态。

**设计风格与灵感:**
严格参考 "The Fork" (thefork.com) 的干净、现代、内容优先、值得信赖的美学风格。

*   **色彩方案:** 主色调 `#006442` (深绿) 用于按钮和链接；强调色 `#FF7A00` (橙色) 用于促销标签；背景使用 `#FFFFFF` 和 `#F8F8F8`；文字使用 `#333333`。
*   **字体:** 使用现代、清晰的无衬线字体，如 "Inter" 或 "Lato"。
*   **布局:** 响应式布局，桌面优先。

---

### **核心UX逻辑: 动态用户旅程**

应用的UI和导航逻辑必须根据用户的订单状态动态调整。定义两种核心用户状态：

1.  **探索状态 (Discovery State):** 用户没有“进行中”的订单。应用的首要目标是帮助用户发现餐厅和下单。
2.  **追踪状态 (Tracking State):** 用户有一个或多个“进行中”的订单（状态为“商家已接单”、“制作中”或“待取餐”）。应用的首要目标是清晰、醒目地展示订单状态。

---

### **全局与动态组件**

*   **全局 Header (导航栏):**
    *   左侧: "Forkly" Logo。
    *   中间: 一个简洁的搜索框，提示文字为 "搜索餐厅名称..."。
    *   右侧: "首页" 链接、"登录 / 注册" 按钮。如果用户已登录，则替换为 "欢迎, [用户名]" 和一个指向“我的订单”页面的头像图标。
    *   效果: 页面向下滚动时，Header应固定在顶部 (Sticky Header)。

*   **全局 Footer (页脚):**
    *   简洁的三栏布局：关于我们 | 联系方式 | 隐私政策。
    *   背景色为深灰色，文字为白色。

*   **全局动态组件: Active Order Status Banner (当前订单状态横幅):**
    *   **触发条件:** 当用户处于“追踪状态”时，此横幅**必须**出现。
    *   **显示位置:** 固定在页面顶部，位于主导航栏下方。在所有页面（**除了“我的订单”页本身**）都可见。
    *   **设计:** 背景色为淡黄色 (`#FFFBEA`)。左侧有时钟图标，中间是核心状态信息 **“您在 [餐厅名称] 的订单正在 [订单状态]...”**，右侧是一个深绿色的 **“查看详情”** 按钮。
    *   **交互:** 点击“查看详情”按钮，立即跳转到“我的订单”页面。
    *   **生命周期:** 当订单状态变为“已完成”或“已取消”后，此横幅自动消失。

*   **全局 Shopping Cart Sidebar (购物车侧边栏):**
    *   默认隐藏或最小化在页面右侧。当用户添加商品时，从右侧滑出并固定。
    *   **内容:** "我的订单"标题、商品列表（名称 x 数量、单项总价、`+` `-` 按钮）、`商品小计`、`总计`。
    *   底部是一个醒目的深绿色按钮 **"去结算"**。

### **页面、跳转逻辑与状态化默认视图**

#### **页面跳转与默认视图逻辑**

*   **对于“探索状态”的用户:**
    *   **默认/入口页面:** **主页 / 餐厅发现页 (`/`)**。
    *   **下单后跳转:** 当用户成功提交订单后，**必须立即跳转到“我的订单”页 (`/orders`)**。

*   **对于“追踪状态”的用户:**
    *   **默认/入口页面:** 当用户访问网站根域名 (`/`) 或重新登录时，系统检测到其有“进行中”的订单，**必须自动重定向到“我的订单”页 (`/orders`)**。
    *   **返回探索:** 用户可通过点击导航栏的“首页”链接，主动进入餐厅发现页。

#### **Page 1: 主页 / 餐厅发现页 (`/`)**
*   **访问逻辑:** “探索状态”用户的默认首页。
*   **Hero Section:** 高质量美食背景图，大标题 "发现并预订您附近的美食"，下方是包含“餐厅名”、“地点”、“日期”的搜索栏。
*   **Restaurant List Section:** 标题 "附近的精选餐厅"，采用响应式网格布局。
    *   **`RestaurantCard` 组件:** 包含封面图、餐厅名称、菜系类型、人均价格、五角星评分和简略地址。鼠标悬浮有轻微放大和阴影效果。点击卡片导航到餐厅详情页。
*   **动态显示:** “追踪状态”的用户访问此页面时，顶部必须显示 **Active Order Status Banner**。

#### **Page 2: 餐厅详情与菜单页 (`/restaurant/:id`)**
*   **Restaurant Header Section:** 餐厅巨幅照片、大号字体的餐厅名称、评分、地址和营业时间。
*   **Menu Section:** 菜单按**菜品分类**分组，每个分类为可折叠区域。
    *   **`DishCard` 组件:** 包含菜品名称、描述、价格，以及一个 **"添加" 按钮** 或 **数量选择器 (`- 1 +`)**。点击“添加”后，商品立即出现在右侧购物车中。
*   **动态显示:** “追踪状态”的用户访问此页面时，顶部必须显示 **Active Order Status Banner**。
*   **交互约束:** 如果用户在`餐厅A`有“进行中”的订单，此时尝试从`餐厅B`添加商品，必须弹出确认模态框提示将开始新订单。

#### **Page 3: 用户中心 / 我的订单页 (`/orders`)**
*   **访问逻辑:** “追踪状态”用户的默认首页。
*   **布局:** 左侧垂直导航 ("个人信息", "我的订单", "地址管理", "退出登录")，右侧内容区。
*   **右侧内容区 (订单列表):**
    *   **当前订单突出显示:**
        *   “进行中”的订单**必须永远置顶显示**。
        *   该订单的 `OrderSummaryCard` 组件应有独特的视觉样式，例如**淡蓝色背景 (`#E6F4FF`)** 和一个醒目的“当前订单”标签。
        *   卡片内包含一个**订单进度条/步骤条 (Stepper)**，可视化地展示状态：`已下单` -> `商家已接单` -> `制作中` -> `待取餐`。
    *   **历史订单:** “已完成”和“已取消”的订单显示在当前订单下方，按时间倒序排列。
    *   **`OrderSummaryCard` 组件:** 卡片顶部显示**餐厅名称**和**订单状态**（使用不同颜色的标签区分），中间是**下单日期**和**订单总金额**，右侧有“查看详情”按钮。

**技术栈提示:**
请使用 **Vue.js** 作为前端框架，并利用 **Vue Router** 的路由守卫 (Navigation Guards) 实现基于用户状态的自动重定向。使用 **Pinia** 或 **Vuex** 进行全局状态管理，在应用启动时获取并存储用户的订单状态，所有页面的动态显示逻辑均依赖此全局状态。将 `RestaurantCard`, `DishCard`, `ShoppingCartSidebar`, `OrderSummaryCard` 等创建为可复用的组件。
